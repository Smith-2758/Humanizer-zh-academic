# deai 目录说明

本目录用于存放各条语料经过去 AI 处理后的结果。

## 当前结构

- 主目录：仅保留每条样本的 `低严肃度`、`中严肃度`、`高严肃度` 三档版本
- `legacy/`：归档早期未区分严肃度的单文件版本，供回看或对照

## 命名规则

- `S1-...-低严肃度.md`
- `S1-...-中严肃度.md`
- `S1-...-高严肃度.md`

其中：

- `S1` 到 `S5` 对应原始语料编号
- 文件中保留背景信息、原始语料路径、去 AI 后文本、修改日志和效果观察

## 使用建议

如果需要横向比较不同风格，优先查看主目录中的三档版本。
如果需要查看最早的处理结果，再进入 `legacy/`。

## 回归验证方法

每次调整 `SKILL.md` 后，优先使用现有 `corpus/` 与 `deai/` 中的样本做人工回归验证。

推荐顺序：

1. 先看同一条样本的 `低 / 中 / 高` 三档是否真正拉开。
2. 再看低严肃度是否明显更自然，而不是只是“更短”或“更口语”。
3. 重点回看实验结果分析段、管理学研究结论段、项目多结论段、功能测试总结段，这几类最容易残留模板。
4. 再记录不同检测平台的判断结果，重点看共性问题，不只看单个平台分数。

## 最小检查表

回归时至少检查以下几点：

1. 结构是否过匀，是否每段都像“总起 + 分析 + 收束”。
2. 句长是否过匀，是否每句都差不多长。
3. 结论句是否过密，尤其是 `从结果来看`、`总体来看`、`可以看出`、`这说明` 这类表达。
4. 是否反复出现 `第一 / 第二 / 第三`、`验证了……可行性`、`为后续研究提供基础`、`后续可从几个方面展开` 这类收束模板。
5. 事实陈述是否被重复翻译成一层抽象意义判断。
6. 低严肃度是否明显更自然，中严肃度是否正式但不过度平滑，高严肃度是否克制但不过度成熟。
