## 项目简介

**Humanizer-zh-academic-student** 是一个专门为了解决大学生学术写作中“AI 塑料味”过重、容易被 AI 相似度检测系统（如知网 AI 检测）判定为机器生成而开发的深度反检测提示词工具。

与普通的“让人话一点”不同，本工具致力于将浮夸、空洞、喜欢卖弄的 AI 学术腔调，转化为**朴实、严谨、真诚且干脆的理工科学生学术写作风格**。

**核心解决痛点：**
- **学术卖弄**：AI 喜欢用晦涩、高大上的词汇包装简单的实验操作。
- **机械句法**：AI 喜欢用可预测的长从句和泛滥的过渡词（如“此外”、“然而”），这是触发检测器的致命特征。
- **虚假深度**：AI 在写结论和展望时，总是生成诸如“深入探索复杂相互作用”的空洞废话。
- **高危排版**：无意义的粗体滥用、内联标题和破折号。

## 核心改造特色（相比原版）

1. **去卖弄化**：强制术语降级，禁止使用“革命性的”、“底层重构”等公关修辞，回归朴实的报告视角。
2. **反判定句法破坏**：要求强制打乱句子长短节奏，切碎复杂从句，暴力拆解机械的过渡词连接，引入人类思维的跳跃感。
3. **真实在场感**：部分翻转极度冷血的被动语态，恢复实验者的“第一人称”或客观主动语态。
4. **针对性极强**：删除了原版中适用于“播客”、“博客”等随性主观的规则，全盘定向至严谨的“学生学术汇报”语境。

## 目录结构

建议的文件夹结构：
```text
humanizer-zh-academic/
├── SKILL.md       # 技能定义文件（核心反检测与降级规则）
└── README.md      # 说明文档
```

## 使用方法

将此文件夹作为 `Skill` 导入支持该协议的 AI 编辑器（如 Claude Code 等），或者直接将 `SKILL.md` 的内容喂给网页版大模型（如 ChatGPT, Claude）作为 System Prompt 即可激活使用。

**调用示例：**

```text
/humanizer-zh-academic 请帮我把下面这段毕业设计日志去 AI 味：

[粘贴你的 AI 生成的、充满套话的实验日志]
```

## 检测的 AI 常见卖弄与机械模式

本工具重点清洗以下几类高危特征：

1. **宏大叙事与夸大意义**（关键转折点、复杂格局、重塑）
2. **生硬转折掩饰逻辑断层**（此外、值得注意的是、深入探讨）
3. **虚假的文献共识**（广泛认为、专家指出）
4. **空洞的展望**（未来必须探索复杂的相互作用）
5. **同义词堆砌与排比**（不仅是...更是...）
6. **AI 专属高频词**（织锦、彰显、赋能、底座）
7. **排版高危特征**（句首无意义粗体、频繁破折号、内联标题式罗列）
