## 项目简介

**Humanizer-zh-academic-student** 是一个面向学生学术写作场景的中文 skill，用于清理 AI 生成文本中的塑料感、卖弄感和机械句法痕迹。

与普通“把话说得更像人”不同，这个 skill 的目标是把学术文本改成更贴近真实学生写作的状态：朴实、严谨、自然，不空泛，不装腔，也不过度口语化。

当前版本不再只是静态提示词，而是一个**轻交互式 skill**：

1. 用户默认先提供待处理原文。
2. Agent 会自动确认 `语气严肃度`。
3. Agent 会自动确认 `角色`。
4. 参数齐全后，再按对应模板执行改写。

---

## 当前能力

本 skill 重点解决以下问题：

1. **学术卖弄**：把“底层重构”“关键转折点”“复杂格局”这类浮夸表达压回朴实学术语言。
2. **机械句法**：拆掉 AI 偏爱的长从句、机械过渡词和重复节奏。
3. **空洞内容**：去除“未来需要深入探索复杂相互作用”这类无信息量套话。
4. **高危排版**：减少破折号、粗体堆叠和机器化列表感。
5. **场景失准**：通过严肃度和角色参数，让课程论文、毕业设计、实验报告、答辩稿写法产生差异。

## 使用方式

将此文件夹作为 `Skill` 导入支持该协议的 AI 编辑器，或直接将 [SKILL.md](./SKILL.md) 作为系统级提示使用。

### 基本用法

用户默认直接贴原文：

```text
/humanizer-zh-academic 请帮我处理下面这段课程论文正文：

[粘贴原文]
```

若用户没有提前说明参数，agent 会自动补问：

```text
语气严肃度选低、中还是高？
```

随后继续询问：

```text
角色选课程论文作者、毕业设计作者、实验报告作者、答辩陈述稿作者，还是自定义？
```

参数确认后，agent 再输出改写结果。

### 直接带参数调用

如果用户一开始就说明了参数，skill 不会重复提问：

```text
/humanizer-zh-academic 请把下面这段文字按高严肃度、毕业设计作者的写法改一下：

[粘贴原文]
```

---

## 严肃度

当前支持三档严肃度：

1. `低`
   - 更自然，保留适度学生写作痕迹
   - 适合平时作业、小论文、阶段总结
2. `中`
   - 克制、清楚、稳妥
   - 适合大多数课程论文和普通实验报告
3. `高`
   - 更正式、更收束、更凝练
   - 适合毕业设计、正式提交稿、摘要和总结

---

## 预设角色

当前支持四类预设角色：

1. `课程论文作者`
   - 偏分析与论述
2. `毕业设计作者`
   - 偏方案、实现、验证与改进
3. `实验报告作者`
   - 偏实验目的、步骤、结果与误差分析
4. `答辩陈述稿作者`
   - 偏正式口头表达，强调顺口和分段清晰

---

## 自定义角色

当用户选择 `自定义` 时，skill 不会无限开放角色设定，而是只补问三个受控信息：

1. 学科背景
2. 求学阶段或经验
3. 文本用途

例如：

```text
请补充这三个信息：学科背景、求学阶段或经验、文本用途。
```

这样做的目的，是把用户需求映射成一个可控写作画像，而不是让输出风格随意发散。

---

## 默认输出

默认输出包含三部分：

1. 参数确认
2. 重写后的学术文本
3. 精简修改日志

评分表不再默认输出。只有当用户明确要求“打分”或“评估”时，才补充评分结果。

---

## 适合处理的文本类型

本 skill 当前最适合以下场景：

1. 课程论文正文
2. 毕业设计说明文字
3. 实验报告
4. 大作业总结
5. 答辩陈述稿

不建议用于：

1. 营销文案
2. 情绪化演讲稿
3. 社交媒体短内容
4. 与学术场景无关的人设化写作
